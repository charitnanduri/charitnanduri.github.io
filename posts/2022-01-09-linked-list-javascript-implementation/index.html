So, what is a Linked List?#

A Linked List is a linear collection of nodes in memory. Each node has a pointer referencing another node. Let me illustrate and describe the Linked List in detail.

The first node which is also called the head represented above contains two things.

Data which contains a value of 1000. A Pointer ( also called a reference) contains the address of the second node and is represented as NEXT (in the picture above).

Every Linked List starts with a HEAD and ends with a TAIL. If the HEAD is NULL then we can say that the Linked List is empty. A TAIL has a pointer value of NULL (indicating that it is the last item in the collection).

Often Linked Lists are compared to arrays, this is because we can store data in arrays in a linear form similar to Linked Lists. The biggest difference is that Linked Lists are not stored in the contiguous form in memory but rather connected to each other by reference. I have listed out key things to keep in mind while using Linked Lists.

Key things to know about Linked Lists#

  • Linked Lists are stored randomly in memory in a non-contiguous form so allocating resources need not be pre-defined.
  • The size of the Linked List is not static which means they can shrink and grow during execution.
  • Inserting and deleting is fast and easy but traversing is slow because of their linear structure
  • A Linked List has an effective way of utilising memory as it can track chunks of memory that is available for allocation.
  • Many important data structures such as stacks, queues, hash tables and graphs are implemented using a Linked List.

insertAtTail(data) {

  // Check: if list is empty then last node will be head
  if (!this.head) {
    this.head = newTail;
  } else {
    // if not empty list, traverse to tail node
    let tail = this.head;
    while (tail.next) {

    tail.next = newTail;

}

class Node {
  constructor(data, next = null) {
    this.data = data;
    this.next = next;
  }
}

class LinkedList {
  constructor() {
    this.head = null;
    this.sizeOfLinkedList = 0;
  }

  // Insert at the beginning of the list
  insertAtHead(data) {
    this.head = new Node(data, this.head);
    this.sizeOfLinkedList++;
  }

  // Insert at the end of the list
  insertAtTail(data) {
    const newTail = new Node(data);

    if (!this.head) {
      this.head = newTail;
    } else {
      let tail = this.head;
      while (tail.next) {
        tail = tail.next;
      }
      tail.next = newTail;
    }
    this.sizeOfLinkedList++;
  }

  // Get node at specific index
  getByIndex(index) {
    if (index < 0 || index >= this.sizeOfLinkedList) {
      return null;
    }

    let current = this.head;
    for (let i = 0; i < index; i++) {
      current = current.next;
    }
    return current;
  }

  // Insert at specific index
  insertAtIndex(index, data) {
    if (index === 0) {
      return this.insertAtHead(data);
    }

    const previous = this.getByIndex(index - 1);
    if (previous === null) {
      return null;
    }

    previous.next = new Node(data, previous.next);
    this.sizeOfLinkedList++;
  }

  // Remove node at specific index
  removeAtIndex(index) {
    if (index === 0) {
      this.head = this.head.next;
      this.sizeOfLinkedList--;
      return;
    }

    const previous = this.getByIndex(index - 1);
    if (previous === null) {
      return null;
    }

    previous.next = previous.next.next;
    this.sizeOfLinkedList--;
  }

  // Print all elements in the list
  printList() {
    let current = this.head;

    if (!this.head) {
      console.log("It is an empty list");
      return;
    }

    const elements = [];
    while (current !== null) {
      elements.push(current.data);
      current = current.next;
    }
    console.log(`Your Linked List: ${elements.join(' -> ')}`);
  }

  // Get the size of the list
  getSize() {
    return this.sizeOfLinkedList;
  }
}

Summary#

Linked Lists are basic yet powerful data structures that can help us define structure complex data. There are two more types of Linked Lists known as Doubly-Linked List and Circularly Linked List.

Further reading into Linked Lists includes time complexity defined by Big O.
